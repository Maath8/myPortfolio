About the Project

This IoT system uses an ESP32 board to retrieve real-time weather data from an online weather service. Based on the received data, the system executes predefined scenarios and automated responses. The ESP32 processes the weather information and performs different actions depending on conditions such as temperature, humidity, or weather status.

🛠️
How It Was Built

  • ESP32 microcontroller programmed via Arduino IDE (C++)
  • Wi-Fi connection to retrieve real-time weather data from an online API
  • Weather data processing on the ESP32 to determine environmental conditions
  • ESP32 WebServer library to host a live dashboard
  • Scenario-based automation where the ESP32 triggers different actions depending on the received weather information

⚡
Challenges We Faced

  • Weather Data Filtering: Retrieving the correct weather information for a specific city from the weather website required parsing and filtering the received data. The solution involved identifying the relevant fields in the response and extracting only the necessary data for the system logic.
  • Wi-Fi Stability: The ESP32 occasionally lost connection to the Wi-Fi network. This issue was addressed by implementing an automatic reconnection mechanism in the main loop to maintain continuous data retrieval from the weather service.

💡
What I Learned

Through this project, I gained experience with HTTP requests to retrieve real-time data from an online weather service. I learned how to perform API integration using the ESP32 and process the received data for system logic. I also implemented IoT automation logic, where the ESP32 executes different scenarios based on the weather conditions.
