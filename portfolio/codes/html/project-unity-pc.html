About the Project

The game idea started when my teammates suggested different types of games. I proposed combining all their ideas into a single project, and we began developing our game around this concept. The game focuses on mission-based gameplay, vehicle mechanics, and interactive environments that provide players with diverse activities across the map. It was developed using modular C# scripts and object-oriented programming principles to ensure clean architecture, scalability, and ease of future expansion.

🛠️
How It Was Built

  • Unity Engine (URP) for rendering, lighting, and open-world scene management
  • C# scripts for player controller, camera, and game logic
  • Vehicle controllers for driving, racing, and transportation systems
  • Physics and collision systems for realistic vehicle and environment interactions
  • Modular architecture using object-oriented programming for scalable game development.

⚡
Challenges We Faced

  • The main map took a long time to make, but the result was worth it.
  • The taxi gameplay algorithm was complex and took 2 days to make.
  • The map was too large to make a mini-map, so we solved it using waypoints.
  • Adjusting the character’s position when entering a vehicle and syncing it with the car was tricky to make.

💡
What I Learned

This project improved my algorithmic thinking and problem-solving skills. Acting as the team leader provided valuable leadership experience. It deepened my understanding of the Unity lifecycle, the separation of game and UI logic, and techniques for optimizing real-time 3D applications. Working with animation state machines and colliders gave me insights into game feel, player experience, and the importance of physics interactions in game development.
